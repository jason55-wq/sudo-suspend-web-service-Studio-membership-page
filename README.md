# 工作室會員管理網站 MVP

這是一個可在本機執行的 Flask + SQLite 版本，重點是：

- 會員可註冊、登入、登出
- 管理員可建立產品與購買紀錄
- 會員登入後只看得到自己已購買的產品
- 下載時由後端檢查權限，檔案不會以公開網址直接暴露

## 專案結構

- `app.py`：Flask 主程式與路由
- `models.py`：SQLite 資料模型
- `extensions.py`：db/login manager
- `config.py`：設定
- `templates/`：HTML 頁面
- `static/style.css`：基本樣式
- `init_db.py`：手動初始化資料庫

## 資料表

會建立 4 個基本資料表：

- `users`
- `products`
- `orders`
- `order_items`

## 安裝

1. 建立虛擬環境
2. 安裝依賴：

```bash
pip install -r requirements.txt
```

## 檔案資料夾

請先確保本機有這個資料夾：

```text
D:\member
```

產品的 `file_path` 會限制在這個資料夾底下，例如：

- `course.zip`
- `subfolder\manual.pdf`

## 啟動

1. 先初始化資料庫：

```bash
python init_db.py
```

2. 啟動網站：

```bash
python app.py
```

3. 打開瀏覽器進入：

```text
http://127.0.0.1:5000
```

## 測試流程

1. 註冊第一個帳號，系統會自動把它設成管理員。
2. 用管理員登入，進入「建立產品」。
3. 輸入產品名稱、說明、以及 `D:\member` 底下的檔案路徑。
4. 到「建立訂單」，選一個會員與一個產品，建立購買紀錄。
5. 用那個會員登入。
6. 在「我的產品」頁面只能看到自己買過的產品。
7. 點「下載」會走後端授權檢查，不會直接開放檔案路徑。

## 下載安全性

下載路由會先檢查：

- 目前登入會員是否真的有買該產品
- 檔案是否存在於 `D:\member`
- 是否是可讀取的實際檔案

如果任一條件不成立，就會拒絕下載。

## MVP 限制

這版先不含：

- 金流串接
- 商品上架審核流程
- 會員分級權限
- API 文件
- 前後端分離

如果你要，我下一步可以幫你加上：

1. 會員手動購買流程
2. 後台管理頁面優化
3. 商品上傳檔案功能
4. 匯出訂單與會員資料
